# Ranges

Ranges is a component for creating and managing multiple range filters. It allows users to select a metric and define its value range — either as free-form numeric input with optional min/max bounds, or using predefined select options.

Key features:

- Add and remove multiple range filters dynamically
- Supports numeric input ranges with configurable min/max bounds
- Supports select-based ranges with predefined ordered options
- Configurable maximum number of applied ranges
- Pre-populate ranges via `initialState`
- Programmatic control via `apiRef` (get, set, reset)
- Built-in validation: boundary checks, required fields, and from ≤ to constraint

## With Api Ref

Demonstrates programmatic control of the Ranges component via the `apiRef` prop. The `RangesApiRef` interface exposes three methods:

- `getRanges()` — returns the current ranges state
- `setRanges(ranges)` — replaces the current state with the provided ranges
- `resetRanges()` — resets ranges back to `initialState` (or empty if not set)

The Reset button in this story calls `apiRef.current?.resetRanges()` to clear all applied ranges.
